# Troubleshooting

## Finder Does Not Open When Creating A Project

Folder selection only works in the installed Craftkeep desktop app. If you are using the browser dev preview, create-project folder selection may not be available.

If you are in the desktop app and Finder still does not open:

1. Quit Craftkeep.
2. Reopen Craftkeep.
3. Try creating the project again.

## Mock Project Is Missing

Open the Help menu and choose `Mock Project`. Craftkeep should restore the bundled demo project and switch to it.

## Settings Do Not Seem To Apply

Try navigating away from Settings and back. If appearance settings still look wrong, use:

`Settings > Advanced > Clear local UI state`

This resets theme, appearance, and sidebar preferences.

## Currency Does Not Change

Currency affects budget and financial displays. Existing text-only notes or manually typed values will not be rewritten.

## The App Looks Too Small Or Too Large

Go to:

`Settings > Appearance > Zoom`

Set the zoom back to `100%` if the app feels mis-scaled.

## Project Files Are Safe After An App Error

Craftkeep stores project data in the project folder on disk. If the app catches an unexpected error, reload the app first. Your project folder should remain intact.
